# Test Commands Reference

## Available Commands

### Run All Integration Tests (Recommended)
```bash
npm run test:integration
```
**What it does:**
- Updates ABIs from contracts
- Starts test orchestrator (spins up anvil, deploys contracts)
- Runs all integration tests
- Cleans up anvil when done

### Run Unit Tests Only
```bash
npm run test:unit
```
**What it does:**
- Runs unit tests only (no blockchain needed)
- No integration tests

### Run Tests in Watch Mode
```bash
npm run test:watch
```
**What it does:**
- Watches for file changes and re-runs tests
- Useful for development

### Run All Tests
```bash
npm run test:all
```
**What it does:**
- Runs unit tests first
- Then runs all integration tests via orchestrator

## Troubleshooting

### Error: "WALLET_PRIVATE_KEY not set"
**Solution:** Run `npm run test:integration` instead of `npm test`

The orchestrator must run first to set up `.env.test` with all environment variables.

### Error: "WEB3PGP_CONTRACT not set"
**Solution:** Same as above - run `npm run test:integration`

### Error: "invalid private key, expected hex or 32 bytes"
**Solution:** Ensure the private key in `.env.test` is in proper hex format:
- Must start with `0x`
- Must be exactly 64 hex characters (32 bytes)

The improved error handling in `test-wallet.ts` will now show exactly what format was received.

## Environment Variables (Auto-Generated by Orchestrator)

When tests run via orchestrator, the following are set in `.env.test`:

```bash
WALLET_PRIVATE_KEY=0x...        # Test wallet private key (32 bytes)
RPC_URL=http://127.0.0.1:8545   # Anvil RPC URL
DEXES_WEB3SIGN=0x...            # Web3Sign contract address
DEXES_WEB3PGP=0x...             # Web3PGP contract address
DEXES_ACCESS_MANAGER=0x...      # AccessManager contract address
```

These are automatically created by `scripts/test-orchestrator.js`.
